The Challenges

ticktBox wanted to create a new feature for studios to create, edit and approve (ROE) Run of Engagement in high efficiency. So we first did research on current methods of creating ROE, and found out the problems existing. After that, we started to think about our own way of making ROE.

How it works?

Design

Flowchart & Contents specification

After specify the features and processes we want to implement, we started to draw user-flow on the chart. We tried to make a simple and clear user flow for studios to create their ROE. However, there are actually so many contents we need to add into each process. So it is challenging to make sure the flows are simple, at the same time keep all the content needed. We used steps by steps methods, at the same time combine similar contents into sections in order to make sense for users. After drawing flowchart, we started a google doc to add details contents we need for each contents. In this way we can have ideas of what we should add in each pages. When we are clear about all the contents, we can start designing the interfaces.

Sketches on paper

Based on details contents we added in google doc, we started to draw sketches of each pages on paper. ROE is one of the features in ticktBox backend services, so we need to keep the consistency of the design. The most important thing we need to work on is the layout of all the contents. Also, we need to make sure the flow is clear. By drawing sketches on paper, we now have ideas of what we should do next.

Design based on original ticktBox UI kit

The next step is to polish the interfaces from scratches. Since ROE is one of the features we wanted to implement to ticktBox backend service, we need to keep the consistency of design. By following the original UI kits, we started to design every pages based on the contents we discussed and the sketches we did.

Feedbacks & Changes

Even though we thought carefully on the contents on every pages, there are still many problems show in our design. We send our design to team leaders for testing and they listed out several problems we need to fix. Based on their feedbacks, we made some changes to improve our design.

Reflection

Designing ROE feature is really a challenging project. There are so many things we need to consider when implementing the functions for users. First of all, we should think carefully about all the information and contents users needed. Since this project is for two different groups of users, publicists and studios. So we should also considered the differences between this two groups of users and what we should add or subtract.

Also, there is one significant thing I learned from this project is the importance of keep consistency in design. Since ROE is one of many features in ticktBox backend, when making the designs, we should follow strictly the style so that users will not get confused when they go to our new features. At the beginning of the design, I tried to make things unique by adding some my own designs. I thought this may be more elegant. However, when we do the testing, we found out that users can easily get confused when they see the different designs, especially for those old users. So in the end, we followed the original UI kits.

Overall, I really learned a lot from this challenging and interesting project!
